# IPTV 电视直播源收集器（改进版）

🚀 收集整理国内可用的电视频道直播源，支持直接订阅使用。改进版解决了原版本频道播放不了的问题。

## 🔍 问题诊断与解决方案

原项目存在的主要问题：
1. **源失效** - 北邮教育网源已无法访问
2. **分类过时** - 频道分类系统不够全面
3. **测试不严格** - 只检查HTTP状态码，不验证流媒体格式
4. **维护不足** - 缺乏定期更新机制

## 📡 使用方法

### 方式一：订阅链接（推荐）
复制以下链接到 IPTV 播放器中订阅：
```
https://chaniug.github.io/IPTV-Collector/iptv-cn.m3u
```

### 方式二：下载 M3U 文件
直接下载 [iptv-cn.m3u](iptv-cn.m3u) 文件（国内源），导入播放器即可。

### 方式三：JSON 格式数据
下载 [channels-cn.json](channels-cn.json) 文件用于开发或其他用途。

## 📱 推荐播放器

- **TVBox** - Android 平台，功能强大
- **PotPlayer** - Windows 桌面端
- **VLC** - 跨平台开源播放器
- **IINA** - macOS 专用

## 📺 频道分类

改进后的分类系统：

- **央视台** - CCTV 1-17 全系列、CETV、CGTN 等
- **卫视台** - 湖南卫视、浙江卫视、江苏卫视、东方卫视、凤凰卫视等全国各大卫视
- **卡通类** - 金鹰卡通、优漫卡通、卡酷少儿等儿童频道
- **新闻类** - 新闻资讯频道
- **体育类** - 体育赛事频道
- **电影类** - 电影、影视剧场频道
- **其他台** - 地方台、专业频道等其他类型

## 🛠️ 本地部署与更新

### 安装依赖
```bash
npm install
```

### 手动更新源
```bash
npm run collect      # 使用实用版采集脚本（推荐）
npm run collect:backup     # 使用原版采集脚本
npm run collect:improved   # 使用改进版采集脚本
npm run collect:practical  # 明确使用实用版
```

### 自动更新并提交
```bash

```

### 本地测试
```bash

# 在浏览器中打开 http://localhost:8080
```

## 🔧 故障排除

### 问题：部分频道无法播放
**可能原因：**
1. 源已经失效 - 直播源的时效性很强
2. 网络限制 - 某些源可能需要特定网络环境
3. 播放器不支持 - 尝试使用不同的播放器

**解决方案：**
1. 运行 `npm run collect` 更新最新的可用源
2. 尝试使用 VPN 或更改网络环境
3. 更换播放器
4. 使用网页版测试工具进行源测试

### 问题：网页显示空白
**解决方案：**
1. 检查 `channels.json` 文件是否存在且格式正确
2. 检查浏览器开发者控制台是否有错误
3. 确保HTTP服务器正常运行

### 问题：频道分类不正确
**解决方案：**
1. 编辑 `collect-practical.js` 文件中的 `categorizeChannel` 函数
2. 在 `channels.json` 中手动调整分类
3. 提交issue报告分类错误的频道
